# S-pot デザイン仕様 v2 — 「旅人きどり」

2026.06.12 / 松本瞬 × Claude

> 「旅の記録を残すアプリ」ではなく
> **「人生で訪れた場所を集め、自分だけの地図を完成させていくアプリ」**

ユーザーが感じるべき感情は「記録しなきゃ」ではなく「**地図を完成させたい**」。
世界観: 古い旅ノート / フィールドノート / 和紙 / 水彩 / フィルム写真 / コーヒー染み / 静かな旅 / 図鑑。SNS感は不要。

---

## 0. 体験の核(行動ループ)

```
旅をする → 写真を選ぶ → 「印を押す」(記録) → 地図に水彩が滲む
    ↑                                            ↓
    └──── 「まだ白い場所」が目に入る ←── 図鑑を眺める
```

設計原則:
- 保存ボタンは「保存」ではなく「**この場所に印を押す**」。行為自体を収集の儀式にする
- 空白(未訪問)は罰ではなく**誘い**。「まだ見ぬ景色が、きっとある。」
- 数字より絵で進捗を見せる。達成率より「色づいた地図」が先

## 3. デザインシステム

### カラートークン
| トークン | 値 | 用途 |
|---|---|---|
| --washi | #F5F0E6 | 背景(全画面) |
| --sumi | #2E2A25 | 文字・線 |
| --koke | #8E9A7A | 第一アクセント。訪問の水彩・進捗 |
| --koke-deep | #6D7B64 | 苔の濃部・ホバー・選択 |
| --tsuchi | #9A7B5F | 第二アクセント。印(スタンプ)・日付・英字キャプション |
| --sumi-faint | #2E2A25 @40% | 補助テキスト |
| --hairline | #2E2A25 @12% | 罫線 |

朱(#B23A24)は引退し、**印は土色(--tsuchi)**に統一。彩度の高い色はゼロにする。
地域カラー6色も廃止 → 訪問の濃淡(苔の重ね塗り)で表現。色数を絞るほど和紙が生きる。

### タイポグラフィ
- 見出し: Shippori Mincho B1(県名32px/ページ題24px、字間0.08-0.14em)
- 本文: Zen Kaku Gothic New 14px / 行間2.0
- 英字キャプション: 10px / 字間0.3em / --tsuchi
- 数字(統計): 明朝22px。数字は常に「12 **ヶ所**」のように単位を小さく添える

### テクスチャ・質感(この世界観の本体)
1. **和紙**: 背景に微細なfeTurbulenceノイズ(opacity 3-4%)
2. **水彩にじみ**: シルエット内の訪問地点からradialGradient+turbulence displacementで滲む(§5)
3. **ポラロイド**: 白フチ12px・下フチ広め・回転-2°〜2°・影は`0 2px 8px rgba(46,42,37,.12)`
4. **印(スタンプ)**: 土色の角丸なし正円、押印アニメ(scale 2.2→1, 回転-6°)は現行を色替えで継承
5. **コーヒー染み**: 装飾SVG(薄い輪じみ)をログ一覧の年区切りなどに控えめに
6. **角丸**: 0(直線基調は継承)。ポラロイドのみ2px

### モーション
- 基本は遅め: 400-700ms ease-out。「静かな旅」なので弾まない
- 水彩の色づき: 記録保存後、その地点から1.2秒かけて滲み広がる(達成の瞬間を儀式化)
- prefers-reduced-motion 対応必須(現行踏襲)

## 4. UX改善案

1. **CTAの言葉を変える**: 「記録する」→「**印を押す**」/「写真から自動記録」→「**写真から旅を引き出す**」
2. **ひとことファースト**: 記録文は「ひとこと(1行)」を必須最小に。長文は詳細画面で「あとから書き足す」。書く重さを最小化
3. **空白の誘い**: 白い県をタップしたとき「東京都は、まだ白いままです」+ 過去に行ったことがあれば「あの日の写真から印を押せます」
4. **行きたい印(下書きピン)**: 行きたい場所をグレーの空印として置ける。訪問して写真を撮ると本物の印に変わる(ロケハン情報と相性◎)
5. **達成の見せ方**: 保存→押印アニメ→シルエットに水彩が滲む→「東京都 34%→37%」と小さく進む。数字は最後
6. **オンボーディング**: 初回はカメラロールから1枚選ばせて30秒で最初の印まで連れて行く

## 5. コンポーネント設計(実装しやすさ優先)

```
app/
 ├ JapanAtlas        … 日本地図(訪問=水彩塗り)※JapanMap改修
 ├ PrefHero          … 県シルエット+水彩にじみ ※PrefArt改修
 │   └ WatercolorFill … SVG: シルエットをclipPathに、記録GPS点に
 │                      radialGradient円(r=記録数で成長)+turbulence
 ├ StatsBar          … 訪問数/達成率/写真枚数(+水彩進捗バー)
 ├ PolaroidCard      … 写真+場所名+日付+ひとこと(回転はid hash で決定的に)
 ├ StampButton       … 主CTA(押印の儀式)
 ├ StampCelebration  … 押印演出(朱→土色に変更)
 ├ BadgeShelf        … 旅章棚(§7)
 └ RecordForm / SpotDetail / SharedFeed / LogList … 現行を配色のみ更新
```

**達成率の定義(実装可能な形)**:
- 県の達成率 = 県のbboxを6×6グリッドに分割し、県内セルのうち記録GPSが入ったセルの割合(GPSなし記録は+1セル扱い)。DB変更不要、クライアント計算で済む
- 全国 = 訪問県数/47
- 水彩の滲み半径 = セル占有に比例。「広く動くほど県が完成する」がそのまま絵になる

**データ**: 現行スキーマで全て成立(records/photos/scout/visibility)。「行きたい印」だけ将来 `records.kind('visit'|'wish')` を追加

## 6. モバイルファースト設計

- 1カラム・縦積み。max-width 560は維持しつつ、スマホでは余白を12pxに圧縮
- 主CTA「＋ 印を押す」は**画面下部固定**(safe-area-inset-bottom対応済みの上に置く)
- タップ目標44px(対応済み)・hover非依存(照準2タップ方式は全地図で実装済み)
- ポラロイドは横スクロール(スナップ)。1.5枚見せて続きを暗示
- 画像: loading="lazy" + 長辺2000pxリサイズ(済)。LCP対策でヒーローは写真でなくSVG
- PWA(Phase 2): ホーム画面追加・オフライン閲覧。スプラッシュ=地球
- 実機確認のため**Vercelデプロイを最優先タスクに**

## 7. ゲーム要素・継続の仕組み

### 集める(コレクション)
- **旅印帳**: 押した印が御朱印帳の見開きに並んでいくページ(月別)
- **景色図鑑**: 海・山・街・夜・橋・社寺・島…のカテゴリ枠。初めて撮ると図鑑のシルエットが埋まる(ポケモン図鑑の「???」方式)
- **県の完成**: 達成率100%で県シルエットが金茶色に「装丁」される

### 旅章(バッジ)
| 旅章 | 条件 |
|---|---|
| 旅のはじまり | 最初の印 |
| 十印 / 五十印 / 百印 | 記録数 |
| 夜の旅人 | 夜景カテゴリ10枚(scout.best_time=夜景) |
| 四季の人 | 同じ県で春夏秋冬の記録 |
| 海を渡る | 初の海外印 |
| 完成の県 | 県達成率100% |
| 四十七 | 全県制覇(金の印) |

### 続けたくなる仕組み(プレッシャーなしで)
- **白地図の便り**(月1通知): 「隣の山梨県が、まだ白いままです」— 催促ではなく地図からの手紙の文体
- **季節の誘い**: 桜・紅葉の時期に「いま色づく県」を地図上でそっと光らせる
- **年の図鑑**: 年末に「2026年の旅」1ページを自動生成(印の数・色づいた県・ベスト写真)。シェアはここだけ(SNS感を持ち込まない逃し弁)
- **ストリークは作らない**: 連続記録の概念は世界観に反するので不採用
